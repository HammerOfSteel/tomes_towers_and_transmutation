# Cave & Glade Integration
> ⚠️ GAP — Place cave and glade entrances on the realm overworld map.

## Status: ⚠️ Not planned

## Goal
Caves and glades generated by OW-C appear as entrance props on the overworld. Caves are in mountain/bog biomes; glades are in forest biomes.

## Tasks

### CG-1 — Cave Entrance Prop
- [ ] `buildCaveEntrance(biome): THREE.Group` — rock arch or hillside opening
- [ ] Biome variants: crystal (glowing), lava (scorched rock), ice (icy arch), fungal (overgrown), ancient (carved)
- [ ] [E] Enter Cave prompt

### CG-2 — Glade Entrance
- [ ] `buildGladeEntrance(): THREE.Group` — mossy stone arch, fairy-ring mushrooms, light shaft
- [ ] Subtle ambient particle effect (fireflies, floating spores)

### CG-3 — Realm Map Placement
- [ ] `RealmData` extended: `caves: Array<{x, y, seed, biome}>`, `glades: Array<{x, y, seed}>`
- [ ] OW-A generator places:
  - Caves: 2-4, prefer mountain/bog/tundra biomes
  - Glades: 1-3, prefer forest biomes
- [ ] Entrances rendered at correct terrain positions

### CG-4 — Scene Transition
- [ ] Same pattern as dungeon: fade → load `CaveScene`/`GladeScene`
- [ ] Return places player at entrance

### CG-5 — Minimap Icons
- [ ] 🌿 icon for glade, 🕳 icon for cave on OWMinimap

## Dependencies
- Requires: OW-C cave generator ✅
- Requires: RI-1 terrain mesh
- Requires: same SceneRouter as dungeon integration
